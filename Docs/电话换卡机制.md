# 聊天事件机制（父母 / 朋友）

## 1. 机制概述

“聊天事件”是玩家通过与不同对象进行对话，消耗手牌并获取关键卡牌的特殊互动机制。

聊天对象分为三类：
- 爸爸
- 妈妈
- 朋友

不同聊天对象对应不同的提问内容、卡牌限制、结算方式与奖励结果：
- 父母聊天偏向“问题对应正确答案”，强调玩家是否能根据问题提交合适卡牌。
- 朋友聊天偏向“自由表达与匹配度判断”，玩家可以使用所有卡牌，但失败风险更高。

---

## 2. 聊天基础流程

每次聊天事件固定包含 **3 轮对话**（朋友聊天为“最多 3 轮”，可能提前结束）。

每一轮流程：
1. 聊天对象提出问题。
2. 玩家从手牌中选择卡牌作答。
3. 系统记录该轮提交的卡牌内容。
4. 进入下一轮或按规则结算。

三轮结束后，根据聊天对象类型进行最终判定。

---

## 3. 父母聊天机制（爸爸 / 妈妈）

父母聊天属于**低风险对话事件**，不会因为单轮回答错误而提前结束。

### 3.1 基础规则

- 每次与爸爸或妈妈聊天时，固定进行 **3 轮**。
- 父母聊天中：
  - 不会提前结束聊天。
  - 不会因为单轮回答错误而吞卡。
- 三轮全部完成后统一结算。

三轮累计提交的卡牌中，至少需要包含：
- 1 张具象卡
- 1 张抽象卡

只要三轮累计同时包含至少一张具象卡和一张抽象卡，即可进入正确率结算。

### 3.2 无效提交规则

如果三轮结束后，累计提交的卡牌中没有同时包含至少一张具象卡与一张抽象卡，则本次聊天视为无效。

无效情况：
- 只有具象卡
- 只有抽象卡
- 没有提交有效卡牌

有效情况：
- 同时包含具象卡和抽象卡

无效时：
- 系统按照玩家提交的卡牌类型进行退还处理，并结束本次聊天结算。

### 3.3 正确率结算

若满足基础提交条件（三轮累计至少包含 1 张具象卡 + 1 张抽象卡），则进入正确率结算。

正确率结果：
- **90% 或以上**：检测玩家当前手牌，给予一张“可能会用到的”抽象卡或具象卡。
- **低于 90%**：聊天完成，但无法获得奖励。

## 4. 爸爸聊天内容

爸爸的问题主要围绕：
- 天气
- 交通工具
- 学到的特别知识

### 4.1 爸爸问题与对应卡牌

#### 第一轮：天气问题
爸爸提问：
> “英国今天天气怎么样呀？”

可对应卡牌：
- 具象卡：雨伞、衣服
- 抽象卡：可提交（用于满足父母聊天的抽象卡累计要求）

#### 第二轮：交通问题
爸爸提问：
> “你是坐什么交通工具出去的？”

可对应卡牌：
- 具象卡：巴士、的士
- 抽象卡：可提交

#### 第三轮：知识问题
爸爸提问：
> “有没有学到什么很特别的知识？”

可对应卡牌：
- 具象卡：醒狮、脸谱、雕像
- 抽象卡：可提交

补充：
- 父母聊天固定三轮，不会提前结束。
- 三轮结束后统一结算。
- 玩家需在三轮累计提交中至少包含一张具象卡和一张抽象卡，并根据问题提交对应卡牌。
- 若三轮正确率达到 90%，可获得对应抽象卡或具象卡奖励。

## 5. 妈妈聊天内容

妈妈的问题主要围绕：
- 饮食是否习惯
- 天气如何
- 是否多喝水
- 是否照顾好自己

### 5.1 妈妈问题与对应卡牌

#### 第一轮：饮食问题
妈妈提问：
> “那边的饮食还习惯吗？”

可对应卡牌：
- 具象卡：蒸笼、牛肉丸、荔枝、碳烤生蚝、白切鸡、乳鸽、蛋挞
- 抽象卡：可提交

#### 第二轮：天气问题
妈妈提问：
> “温度怎么样？”

可对应卡牌：
- 具象卡：雨伞、衣服
- 抽象卡：可提交

#### 第三轮：关怀问题
妈妈提问：
> “有没有多喝水呀？”

可对应卡牌：
- 具象卡：水杯
- 抽象卡：可提交

## 6. 朋友聊天机制

朋友聊天是开放式高风险对话事件。

与父母聊天不同：
- 不设置固定标准答案。
- 玩家可以使用所有卡牌作答。
- 核心判断是“提交卡牌是否自然回应问题”。

### 6.1 基础规则

- 每次朋友聊天**最多进行 3 轮**。
- 所有卡牌都可作为回答。
- 每轮都会即时判断匹配度。
- 若任意一轮匹配度低于 **70%**，聊天会提前结束。
- 若提前结束，本轮提交卡牌会被吞掉。
- 若三轮都顺利完成，玩家获得最终融合卡。

### 6.2 问题内容（示例）

朋友问题偏日常聊天，例如：
- “你那边现在几点？”
- “天气怎么样？”
- “有没有吃到什么好吃的？”
- “有没有什么有趣的事情？”

### 6.3 卡牌使用范围

朋友聊天中，玩家每轮投入至少 **2 张卡牌**，且可使用所有类型卡牌：
- 天气类
- 食物类
- 交通类
- 文化类
- 情绪类
- 抽象卡
- 具象卡
- 融合相关卡牌

示例：
- 问题：“有没有吃到什么好吃的？”
  - 可提交：蛋挞 + 抽象卡
  - 可提交：牛肉丸 + 碳烤生蚝

### 6.4 匹配度规则

朋友聊天采用匹配度而非标准答案判断：
- **90% 或以上**：回答非常自然，进入下一轮
- **70%–89%**：回答基本合理，进入下一轮
- **低于 70%**：吞掉本轮卡牌，并提前结束聊天

### 6.5 失败处理

若任意一轮匹配度低于 70%，则本次朋友聊天立即失败：
- 本轮提交卡牌被吞掉
- 聊天提前结束
- 后续轮次不再进行
- 本次聊天无法获得融合卡

### 6.6 成功奖励

若玩家成功完成三轮聊天，且每轮匹配度都达到 70% 或以上：
- 玩家获得一张最终所需的融合卡

融合卡示例：
- 异乡日常
- 文化连接
- 被理解的瞬间
- 跨文化记忆
- 朋友的回应
- 共同话题

---

## 7. 父母聊天与朋友聊天区别

| 对比项 | 父母聊天 | 朋友聊天 |
|---|---|---|
| 对话轮数 | 固定 3 轮 | 最多 3 轮 |
| 是否提前结束 | 不会提前结束 | 会提前结束 |
| 判断方式 | 三轮结束后统一计算正确率 | 每轮即时计算匹配度 |
| 卡牌限制 | 需要根据问题提交对应卡牌 | 所有卡牌都可以使用 |
| 卡牌类型要求 | 三轮累计至少 1 张具象卡 + 1 张抽象卡 | 不强制类型，但需要符合问题 |
| 每轮投入要求 | 建议/规则中可设每轮至少 2 张卡牌 | 每轮至少 2 张卡牌 |
| 失败处理 | 按提交卡牌类型退还处理 | 低于 70% 时吞掉本轮卡牌 |
| 成功条件 | 三轮正确率达到 90% | 三轮匹配度均达到 70% |
| 成功奖励 | 获得对应抽象卡或具象卡 | 获得最终融合卡 |
| 机制主题 | 家庭关心、稳定回应 | 自由交流、风险表达 |

---

## 8. 正式规则总结

玩家可以通过聊天事件与爸爸、妈妈或朋友进行三轮对话。

父母聊天为固定三轮，不会提前结束。玩家需要根据父母提出的问题提交对应卡牌，并在三轮累计提交中至少包含一张具象卡和一张抽象卡。三轮结束后，系统根据回答与问题的匹配程度计算正确率。若正确率达到 90% 或以上，检测玩家已有卡牌，给予需要的一张对应抽象卡或具象卡；若未满足基础提交条件，则按照提交卡牌类型退还处理。

朋友聊天为开放式对话，玩家可以使用所有卡牌作为回答。每一轮系统都会即时判断卡牌与问题之间的匹配度。若任意一轮匹配度低于 70%，则本轮提交卡牌被吞掉，并提前结束聊天。若玩家顺利完成三轮对话，且每轮匹配度均达到 70% 或以上，则可以获得一张最终所需的融合卡。
