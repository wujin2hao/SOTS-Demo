# GDD v0.3 —《乱码与卡牌 / Static & Signal》

> **v0.3 修订说明**
> 在 v0.2 基础上做两项核心调整：
> 1. **作业评分系统重写**：去掉 SotS 式边缘符号匹配（与卡牌视觉不符），改为"brief 主题命中 + 卡牌类型多样性"双维度评分
> 2. **电话场景升级为文化冲突的机制载体**：电话不再只是免费产卡，而是要求玩家从牌库分享抽象/融合卡——纯母语 deck 在电话场景失败
>
> 主题、情感底色、卡牌三态视觉、三结局结构维持不变。

> **v0.3.1 原型迭代修订（2026-04-26）**
> 以下三项调整来自第一轮机制原型 playtest 前的范围收窄：
> 1. **牌库容量 12 → 10**：8 张母语卡 + 2 个空位。缩减后每日获卡压力更明显，丢牌决策更早到来。
> 2. **日循环 7 天 → 5 天**：vertical slice 周期缩短，单次游玩节奏更紧凑；完整版相应调整为 3 × 5 天。
> 3. **合成操作改为玩家手动选牌**：点击"Synthesize"后弹出选牌弹窗，玩家自行指定 1 张抽象卡 + 1 张具象卡组合。**配对有限制**：具象卡的隐藏 tag 必须与抽象卡的符号一致才能融合（例：tag=◻ 的具象卡只能与 ◻ 抽象卡配对）。不匹配的卡在弹窗中变灰不可点选。后期可通过配表进一步限制或扩展合法组合。

## 1. 核心概念（30 秒版）

你是一个刚到英国的研究生。每天 **15 点精力**。

上午必须去上课——导师讲话是逐词滚过的乱码，**30 秒实时倒计时**，你只能捕获其中几个词。
下午地图打开，你自己决定剩余精力怎么花：去公园观察、上网搜索、给家里打电话，或者回房间把素材合成成卡牌。
每周交一次作业，按 brief 主题组牌。
听课捕获越多 → 抽象卡越多；打电话越多 → 具象卡越多。
**牌库容量恒定。有些东西必须被丢掉。**
**纯母语 deck 在作业上失败；纯外语 deck 在电话上失败。中间路线最累。**

## 2. 日循环结构

每天两段：

```
[早上 09:00] ─── 强制上课（30 秒）
                       │
                       ▼
              ─── 地图打开 ───
                       │
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
    [Apartment]     [Park]          [School]
                       │
                精力耗尽 / 主动睡觉
                       │
                       ▼
                  [次日 09:00]
```

**精力归零 或 玩家主动"睡觉"** → 推进到下一天。

## 3. 上午：听课（强制 / 30 秒）

```
┌──────────────────────────────────────┐
│  [⏱ 00:30]                            │
│                                       │
│   👨‍🏫    ┌──────────────────────┐    │
│   导师    │ @#￥%……  &*￥#@  ……%& │   │
│           │  ￥#@  *&……%   *@%   │    │
│           └──────────────────────┘    │
│                                       │
│                          🧑 ●●●●●     │
│                          (玩家精力)    │
└──────────────────────────────────────┘
```

**机制**：
- 进入课堂自动消耗 **3 点精力**（出场费，无法回避）
- 30 秒倒计时内，乱码词在对话气泡里**逐词滚过**
- 玩家点击某个乱码词 → "捕获"它 → 消耗 **1 点精力** → 直接转化为 1 张抽象卡（无独立翻译步骤）
- 倒计时结束 或 玩家精力耗尽 / 主动跳过 → 课程结算

**典型一节课**：玩家捕获 5–8 个词，消耗 8–11 点精力，下午剩 4–7 点。

### 3.1 一个待 playtest 的关键设计问题

> **玩家凭什么知道该捕获哪个词？**

三个候选方案，倾向方案 C：

- **方案 A（纯随机）**：所有乱码词等价。玩家瞎猜。→ 破坏玩家能动性，不推荐。
- **方案 B（明示标注）**：重要词高亮显示。→ 破坏"听不懂"的体验内核。
- **方案 C（弱暗示）**：部分乱码词有**微弱视觉差异**（略大、略亮、出现略慢），但暗示**不完全可靠**——有时高亮的也是垃圾词。**玩家在游戏过程中逐渐学会读这些暗示**，命中率提升不是靠角色升级，而是**靠玩家自己习得模式**。

方案 C 让"语言习得"这件事真的发生在玩家身上，而不是被参数模拟出来。但需要 playtest 调暗示强度。

## 4. 下午：地图系统

地图层级（参考玩家草图）：

```
[Map]
 │
 ├── 🏫 School         上午已访问，下午可选择性访问
 │
 ├── 🏠 Apartment      回家
 │    ├── 📞 Telephone     与家人通话
 │    ├── 💻 Internet      上网搜索灵感
 │    └── 📐 Homework      合成卡牌 & 做作业
 │
 └── 🌳 Park           户外
      ├── 🌊 Lake          安静、自然、内省
      └── 🌿 Lawn          社会、活跃、人群
```

### 4.1 各地点行为

| 地点 | 精力 | 产出 | 备注 |
|---|---|---|---|
| 📞 Telephone | **0** | 1 张具象卡（如能"分享"成功，见 4.3） | 唯一免费行为，但有失败条件 |
| 💻 Internet | 2 | 1 张具象卡（数字图像、网络文化） | 信息量大但浮于表面 |
| 📐 Homework — 合成 | **5** | 把 1 张抽象卡 + 1 张具象卡 → 1 张融合卡 | 最贵；玩家手动选择配对；**具象卡 tag 必须与抽象卡符号一致**（后期可配表扩展） |
| 📐 Homework — 出牌 | 0 | 提交作业 | 仅在作业截止日可用 |
| 🌊 Lake | 2 | 1 张具象卡(自然、水、安静意象) | |
| 🌿 Lawn | 2 | 1 张具象卡(人物、社交、城市意象) | |

### 4.2 精力账本（典型一日）

```
开局:                  15 点
强制上课:              -3 点  → 12 点
课堂捕获 5 词:         -5 点  →  7 点
回家上网搜索:          -2 点  →  5 点
打电话:                 0 点  →  5 点
去公园 Lake:           -2 点  →  3 点
合成融合卡(部分):      -3 点  →  0 点
睡觉
```

### 4.3 电话场景：文化冲突的机制呈现

**电话场景的核心机制不是产卡，是承担"和家人渐行渐远"这一主题。**

每次拨通电话，家人会问起你**学校的事**——画面出现一个询问气泡（"你最近学什么了？""你那个老师怎么样？"），并显示一个**"分享请求"指示符 📚**。

**玩家需从当前牌库里"分享"一张卡作为回应。**

| 玩家牌库状态 | 结果 |
|---|---|
| 拥有抽象卡 / 融合卡 / 具象卡 | 玩家选择 1 张抽象卡分享 → "妈也听不太懂……算了你忙吧。" → **不产出任何卡**（短镜头：屏幕另一端的沉默 → 镜头切回房间） |
| 拥有抽象卡 / 融合卡 / 具象卡 | 玩家选择 1 张具象卡分享 → "这个妈妈好像在国内也见过balabala" → **产出 1 张新的具象卡**（一段被家人珍藏的瞬间） |
| 拥有抽象卡 / 融合卡 / 具象卡 | 玩家选择 1 张融合卡分享 → "妈也听不太懂……但是看起来也是蛮好的。" → **不产出任何卡**|
| 牌库内任何卡 | 玩家选择不分享 → "算了你忙吧。" → **不产出任何卡** → 短镜头：屏幕另一端的沉默 → 镜头切回房间 |

**设计内涵**：

电话仍然 **0 精力消耗**——它的代价从来不在精力，**在牌库构成**。

这与作业系统形成**双向夹击**：

- **作业要求多样性（含具象卡）** → 纯抽象 deck 在作业上失败
- **电话要求至少一些抽象/融合卡** → 纯母语 deck 在电话上失败
- 中间路线最累，但唯一两边都不失败

> **注意分享出去的抽象/融合卡不会被消耗**——只是"展示"给家人。但分享会让玩家**意识到**自己的 deck 里这类卡的稀缺。

## 5. 卡牌三态

| 类型 | 来源 | 视觉特征 | 使用 |
|---|---|---|---|
| **抽象卡** | 课堂捕获 | 几何（◻️ ⚪ △ ◇） | 作业主题命中 / 电话分享 |
| **具象卡** | Telephone / Internet / Park | 写实形象（🐶 🍊 🐱） | 作业多样性 / 不能用于电话分享 |
| **融合卡** | Homework 合成（抽象 + 具象） | 由具象演化的几何（"几何狗"） | 双属性，作业里以一抵二 / 也可用于电话分享 |

**视觉机制即叙事**：外部学术世界给你**空的几何外壳**，自己的生活给你**温暖的具象**。创作的工作是**把后者变成前者的形态**——但消化过程中，温度有没有被保留？

## 6. 牌库与作业

### 6.1 牌库恒定

- 总容量 **10 张**
- 初始牌库：8 张母语具象卡（玩家从家乡带来）+ 2 个空位
- 新增卡 → 必须从现有牌库选一张**丢弃 / 保留 / 融合**

### 6.2 作业评分：主题命中 + 多样性

每次作业 brief 指定一个**主题符号**（例如：本周作业主题是 ◻️）。
玩家提交 **4 张卡**（数量可在 playtest 中调整）。

#### A. 主题命中（Theme Match）

提交卡中**至少 2 张**与主题符号相关：

- **抽象卡** → 直接看符号是否匹配
- **具象卡** → 每张具象卡有隐藏的"主题 tag"（湖→ ◻️、橘子→ ⚪、猫→ △……）
- **融合卡** → 因为同时含具象基底和抽象符号，**只要其抽象符号匹配主题**即算命中

#### B. 多样性（Diversity）

4 张卡覆盖的"卡牌类型"数量：

- **4 张同类型** → 单调（最低）
- **2 + 2**（抽象+具象 / 抽象+融合 / 具象+融合）→ 有结构（中）
- **1 + 1 + 2 或 全三类** → 综合（最高）

#### C. 关键规则：融合卡的双属性

**融合卡同时计入"抽象类"和"具象类"两个统计。**

这让融合卡在多样性维度上**以一抵二**，但**不能完全取代两侧**。看下表：

| 提交组合 | 主题 | 多样性 | 教授评价 |
|---|---|---|---|
| 4 抽象 | 命中 | 单调 | "Technically on-brief, but where's the work?" |
| 4 具象 | 失败（无抽象主题符号） | 单调 | "I don't see how this engages with the brief" |
| 2 抽象 + 2 具象 | 命中 | 中 | "Some engagement, but feels collaged" |
| 1 抽象 + 1 具象 + 2 融合 | 命中（多个） | 高 | "**A clear voice working through the brief**" ← 最高分 |
| 4 融合卡 | 命中 | **单调（仍是单一类型）** | "Strong voice but underdeveloped — where's the rigor?" ← **故意压低** |

**全融合策略被故意压低**——理由是缺少 raw 抽象/具象的对照。这避免"全融合最优"的策略坍缩。

### 6.3 brief 模糊机制

如果听课捕获不足，brief 的主题符号会有部分以 "?" 显示——玩家被迫赌：

- 猜对了 → 主题命中加分
- 猜错了 → 主题命中失败，且全部命中分数清零

这对应"没听懂作业要求就开始做"的真实赌博感。

## 7. 三结局

通关时由**牌库构成**决定，**无优劣关系**：

- **Clean Slate**（同化）：母语具象卡 ≤ 2，无融合卡 → 作业评价高，**电话连续失败**。结尾镜头：家里来电话，"我现在不太方便，晚点打回去"，挂断，flash 闪过被丢弃的母语卡。
- **Untranslated**（坚守）：母语具象卡 ≥ 6，抽象/融合卡 ≤ 2 → **作业勉强及格，电话也接不下去**。结尾镜头：自己说母语清晰，走廊里同学说话仍是乱码；电话铃响，犹豫片刻，没接。
- **Fluent in Two Tongues**（融合）：融合卡 ≥ 3 → 双语流利，作业和电话都能维持，但**这条路最累**。结尾镜头：熬夜蒙太奇 + 提交作业 + 揉太阳穴 + 接起电话用母语笑着说话。

三个结局演出长度、配乐、镜头语言**等量齐观**。**不奖励"正确路线"**。

## 8. 总流程

| 版本 | 长度 | 用途 |
|---|---|---|
| **Vertical Slice** | 1 周（5 天）+ 1 次作业 | 申请用 demo，约 15 分钟 |
| **完整版** | 3 周（15 天）+ 3 次作业 + 终评 | 约 40–50 分钟 |

vertical slice 必须呈现：
- ✅ 完整的日循环（早上听课 + 下午地图）
- ✅ 至少 1 次作业的"主题 + 多样性"评分
- ✅ 至少 1 次电话场景（成功 或 失败 都展示一次更好）
- ✅ 至少 1 次"丢弃 / 保留 / 融合"的牌库决策
- ✅ 至少 1 次合成（让玩家亲手做出 1 张融合卡）

## 10. 简化追溯（v0.1 → v0.2 → v0.3）

为让 demo 可实现且评分系统经得起策略推演，以下机制已被移除或推迟：

| 机制 | 砍除版本 | 原因 |
|---|---|---|
| 注意力池**动态扩容** | v0.2 | 简化为固定 15 点/天 |
| **5 个语义维度**（Personal/Conceptual/Material/Reference/Context） | v0.2 | 保留 5–6 个抽象符号但不细分语义层 |
| **超市**作为独立场景 | v0.2 | 合并入"上网搜索" |
| **多周长流程** | v0.2 | vertical slice 仅做 1 周 |
| **记忆卡 / 观察卡** 子类型区分 | v0.2 | 简化为统一"具象卡" |
| **SotS 式边缘符号匹配** | v0.3 | 与卡牌视觉不符；改为 brief 主题 + 类型多样性双维度 |
| **三轴评分**（Legibility/Voice/Coherence） | v0.3 | 简化为主题命中 + 多样性两维度 |
| **课堂捕获 + 独立翻译**两步流程 | v0.3 | 合并为"捕获即翻译"一步 |

如 vertical slice 验证机制成立，可在完整版重新引入。

## 11. 申请文档配套（重要）

提交研究生作品集时，**配套设计文档比游戏本身更值钱**。文档应主动讨论：

1. **个人经验 → 机制的转译过程**
   "国际学生作业焦虑"如何具体转化为"30 秒乱码倒计时 + 牌库恒定容量 + 电话失败机制"？为什么是这些机制而不是别的？

2. **与参考作品的关系**
   - 与 *Signs of the Sojourner*：保留了牌库演化（成长伴随失去），**舍弃了边缘符号匹配**——为什么？因为我的卡牌视觉不支持，且我的核心张力来自 deck 构成的双向夹击，不来自单次出牌的拼接技巧
   - 与 *Chants of Sennaar*：保留了在不确定中构建语义理解，舍弃了主动解谜的从容感

3. **失败的中间版本**
   - "拼图比喻"为什么不够好？
   - 为什么放弃了 21 天 roguelike 框架？
   - 为什么把超市并入上网？
   - 为什么把"全 X 卡 → 低分"的简单逻辑改成"主题 + 多样性"双维度？（提示：策略坍缩问题）
   - 为什么电话不是单纯的"免费产卡"？

> 展示 critical design literacy 是 MA 申请最值钱的部分。**敢于在文档里写"我砍掉了什么、为什么砍"，比假装一切都设计完美更显成熟**。

## 12. 待 v0.4 考虑的事项（仅记录，不做）

- **电话场景的反向冲突**：家人偶尔问家常事（"你爸新养了一只猫"），玩家需要具象卡回应。如果 deck 全是抽象卡也会失败。这是双向对称版本，但增加 scope，建议在 vertical slice 验证主向后再加。
- **多个导师 / 同学**：每个 NPC 偏好的 brief 主题不同，玩家在多个对象间分配注意力。
- **作业反馈影响下一周 brief**：动态难度调整。
- **天气 / 心情**：影响每日精力上限。
