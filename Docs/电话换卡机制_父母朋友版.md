# 中英同事件：具象卡 / 抽象卡 / 融合卡（电话换卡版）

## 核心主线

- 外部系统（外）：上课、brief、deadline，主要产出**抽象卡 X**。
- 内部经验（中）：电话、宿舍、日常，主要产出**具象卡 Y**。
- 转译层：Homework 将 X 与 Y 合成为**融合卡 X·Y**（或失败卡）。

---

## 电话事件重构（基于“广东生活习惯文化”）

电话系统不再按 A/B/C 事件分组，而是按**通话对象**分为三类：

1. **和爸爸打电话**：偏工作/学习主题。
2. **和妈妈打电话**：偏生活主题。
3. **和朋友打电话**：工作、学习、生活都可聊。

### 1) 和爸爸打电话（工作/学习向）

- 开启条件：每轮对话需至少投入 **2 张卡** 才能进入换卡结算。
- 轮次上限：单次通话最多 **3 轮**。
- 推荐出牌：
  - 抽象卡 X（课程、方法、项目进度）
  - 融合卡 X·Y（把学习内容说回具体情境）
- 结算逻辑：
  - 爸爸**听懂度高**：该轮有概率产出融合卡（X·Y）。
  - 爸爸**听懂度低**：该轮产出倾向为具象卡 Y 或抽象卡 X。
- 概率与消耗规则（父母通话共用）：
  - 产出融合卡概率固定为 **15%**。
  - 若本轮结果为融合卡：**消耗本轮投入的全部卡片**（高回报高代价）。
  - 若本轮结果为具象卡或抽象卡：按**平换**结算（投入与产出等量对冲，不额外吞卡）。

### 2) 和妈妈打电话（生活向）

- 开启条件：每轮对话需至少投入 **2 张卡** 才能进入换卡结算。
- 轮次上限：单次通话最多 **3 轮**。
- 推荐出牌：
  - 具象卡 Y（饮食、天气、作息、情绪）
  - 融合卡 X·Y（把日常体验与学习压力连接起来）
- 结算逻辑：
  - 妈妈**听懂度高**：该轮有概率产出融合卡（X·Y）。
  - 妈妈**听懂度低**：该轮产出倾向为具象卡 Y 或抽象卡 X。
- 概率与消耗规则（父母通话共用）：
  - 产出融合卡概率固定为 **15%**。
  - 若本轮结果为融合卡：**消耗本轮投入的全部卡片**（高回报高代价）。
  - 若本轮结果为具象卡或抽象卡：按**平换**结算（投入与产出等量对冲，不额外吞卡）。

### 3) 和朋友打电话（全主题向）

- 开启条件：任意 **1 张卡** 即可发起聊天。
- 轮次上限：单次通话最多 **5 轮**。
- 朋友兴趣机制：朋友可能因“不感兴趣”随时终止聊天。
- 首张卡建议与继续聊天概率：
  - 首张为融合卡 X·Y：继续聊天概率 **85%**
  - 首张为具象卡 Y：继续聊天概率 **55%**
  - 首张为抽象卡 X：继续聊天概率 **45%**
- 话题统一性要求：后续出牌与首轮主题相关性需达到 **70%**，否则朋友兴趣快速下降。
- 结算逻辑：
  - 朋友**听懂度高**：每轮可稳定产出融合卡（X·Y）。
  - 朋友**听懂度低**：可能产出具象卡/抽象卡，或直接换卡失败。

---

## 统一换卡结算框架

### 输入
- 当前通话对象（爸爸 / 妈妈 / 朋友）
- 本轮投入卡牌数量
- 本轮卡牌类型组合（X / Y / X·Y）
- 听懂度（由卡牌匹配度 + 话题相关性共同决定）

### 输出
- 成功高：产出融合卡 X·Y（父母通话时触发“吞投入卡”规则）
- 成功中：产出具象卡 Y 或抽象卡 X（平换）
- 成功低：换卡失败（无产出）

### 听懂度建议计算（便于后续程序化）

`听懂度 = 0.5 * 话题相关性 + 0.3 * 卡牌类型匹配 + 0.2 * 对象偏好匹配`

- 话题相关性：与当前通话主线的一致程度（0-100）。
- 卡牌类型匹配：当前对象偏好的卡类型是否充分（0-100）。
- 对象偏好匹配：爸爸偏 X/X·Y，妈妈偏 Y/X·Y，朋友偏首轮兴趣 + 连续一致性（0-100）。

---

## 与合成系统接口

- `contour == abstract.sym` → 产出融合卡 X·Y。
- `contour != abstract.sym` → 产出失败卡（强行套壳，作业扣分）。

电话换卡的设计目的：
- 通过“对象化沟通”提高或降低听懂度；
- 让融合卡产出与真实沟通质量绑定；
- 使“父母定向沟通”和“朋友兴趣沟通”形成两种不同策略压力。
