# GDD v0.5 —《乱码与卡牌 / Static & Signal》

> **v0.3 修订说明**
> 在 v0.2 基础上做两项核心调整：
> 1. **作业评分系统重写**：去掉 SotS 式边缘符号匹配（与卡牌视觉不符），改为"brief 主题命中 + 卡牌类型多样性"双维度评分
> 2. **电话场景升级为文化冲突的机制载体**：电话不再只是免费产卡，而是要求玩家从牌库分享抽象/融合卡——纯母语 deck 在电话场景失败
>
> 主题、情感底色、卡牌三态视觉、三结局结构维持不变。

> **v0.3.1 原型迭代修订（2026-04-26）**
> 以下三项调整来自第一轮机制原型 playtest 前的范围收窄：
> 1. **牌库容量 12 → 10**：8 张母语卡 + 2 个空位。缩减后每日获卡压力更明显，丢牌决策更早到来。
> 2. **日循环 7 天 → 5 天**：vertical slice 周期缩短，单次游玩节奏更紧凑；完整版相应调整为 3 × 5 天。
> 3. **合成操作改为玩家手动选牌**：点击"Synthesize"后弹出选牌弹窗，玩家自行指定 1 张抽象卡 + 1 张具象卡组合。**配对有限制**：具象卡的隐藏 tag 必须与抽象卡的符号一致才能融合（例：tag=◻ 的具象卡只能与 ◻ 抽象卡配对）。不匹配的卡在弹窗中变灰不可点选。后期可通过配表进一步限制或扩展合法组合。

> **v0.4 机制修订（2026-04-26）**
> 本版本对合成系统做核心重设计，引入失败产出路径与扩展符号集：
> 1. **抽象符号从 4 扩展到 6**：新增 ✚（Structural Cross）和 ⬡（Cellular Pattern），共 ◻ ⚪ △ ◇ ✚ ⬡ 六种。
> 2. **具象卡新增"形态轮廓"字段（contour）**：contour 是具象卡的隐藏形态属性，默认与 tag 相同，后期可独立配表。contour 是合成匹配的依据，tag 仍只用于作业主题评分。
> 3. **合成系统：从"匹配才可选"改为"任意可选，结果不同"**：移除弹窗中的变灰限制，所有卡对均可点选。若 concrete.contour = abstract.sym → 产出融合卡（正常路径）；若不匹配 → 产出强行套壳卡（失败卡，−10 pts）。
> 4. **作业评分：命中阈值从 ≥2 调整为 ≥3**，且提交手牌中每张失败卡额外扣 10 分。
> 5. **起始牌库预置 1 张融合卡**（🍊 外婆的厨房 + ⚪ Negative Space），展示成功合成样本；牌库从 9/10 开始。

> **v0.5 机制修订（2026-05-06）**
> 本版本对**电话**与**上网**两个场景做核心重设计，把"中英同事件"作为贯穿主线显式化：
> 1. **电话场景：从"分享卡"重构为"中英同事件 × 出牌结算"**——引入事件 A/B/C（排队 / 请客 vs AA / 节日翻译）。每个事件呈现同一日常情境在中英语境下的语义差，配套具象 Y、抽象 X、融合 X·Y 三态卡的示例化产出；玩家选事件 → 家人提问 → 出 1 张卡 → 按卡类型分支结算。
> 2. **上网行为：从单次 2 精力换 1 张具象卡升级为"独立检索小游戏"**——每次需投入 2–5 张卡作为"检索包"，结果三分（命中复合卡 / 全损 / 偏航成普通抽象卡）；引入 ForeignNoise 跨语境干扰参数与玩家可见的概率/风险预估。
> 3. **主线显式化**：外部系统（课堂、上网）→ 主要产出**抽象卡 X**；内部经验（电话、宿舍、日常）→ 主要产出**具象卡 Y**；Homework 转译层把 X 与 Y 合成为**融合卡 X·Y**（或失败卡）。所有产出渠道围绕这一主线对齐。

## 1. 核心概念（30 秒版）

你是一个刚到英国的研究生。每天 **15 点精力**。

上午必须去上课——导师讲话是逐词滚过的乱码，**30 秒实时倒计时**，你只能捕获其中几个词。
下午地图打开，你自己决定剩余精力怎么花：去公园观察、上网搜索、给家里打电话，或者回房间把素材合成成卡牌。
每周交一次作业，按 brief 主题组牌。
听课捕获越多 → 抽象卡越多；打电话越多 → 具象卡越多。
**牌库容量恒定。有些东西必须被丢掉。**
**纯母语 deck 在作业上失败；纯外语 deck 在电话上失败。中间路线最累。**

## 3. 上午：听课（强制 / 30 秒）

```
┌──────────────────────────────────────┐
│  [⏱ 00:30]                            │
│                                       │
│   👨‍🏫    ┌──────────────────────┐    │
│   导师    │ @#￥%……  &*￥#@  ……%& │   │
│           │  ￥#@  *&……%   *@%   │    │
│           └──────────────────────┘    │
│                                       │
│                          🧑 ●●●●●     │
│                          (玩家精力)    │
└──────────────────────────────────────┘
```

**机制**：
- 进入课堂自动消耗 **3 点精力**（出场费，无法回避）
- 30 秒倒计时内，乱码词在对话气泡里**逐词滚过**
- 玩家点击某个乱码词 → "捕获"它 → 消耗 **1 点精力** → 直接转化为 1 张抽象卡（无独立翻译步骤）
- 倒计时结束 或 玩家精力耗尽 / 主动跳过 → 课程结算

**典型一节课**：玩家捕获 5–8 个词，消耗 8–11 点精力，下午剩 4–7 点。

### 3.1 一个待 playtest 的关键设计问题

> **玩家凭什么知道该捕获哪个词？**

三个候选方案，倾向方案 C：

- **方案 A（纯随机）**：所有乱码词等价。玩家瞎猜。→ 破坏玩家能动性，不推荐。
- **方案 B（明示标注）**：重要词高亮显示。→ 破坏"听不懂"的体验内核。
- **方案 C（弱暗示）**：部分乱码词有**微弱视觉差异**（略大、略亮、出现略慢），但暗示**不完全可靠**——有时高亮的也是垃圾词。**玩家在游戏过程中逐渐学会读这些暗示**，命中率提升不是靠角色升级，而是**靠玩家自己习得模式**。

方案 C 让"语言习得"这件事真的发生在玩家身上，而不是被参数模拟出来。但需要 playtest 调暗示强度。

### 4.3 电话场景：中英同事件 × 出牌结算

**电话场景的核心机制不是产卡，是承担"和家人渐行渐远"这一主题。**

电话仍然 **0 精力消耗**——它的代价从来不在精力，**在牌库构成与手牌组成**。它与作业系统形成**双向夹击**：

- **作业要求多样性（含具象卡）** → 纯抽象 deck 在作业上失败
- **电话要求至少一些抽象/融合卡 / 出得起手牌** → 纯母语 deck 在电话上失败
- 中间路线最累，但唯一两边都不失败

#### 4.3.1 中英同事件：A / B / C

电话不再是无差别"分享一张卡"。每次拨通电话，玩家**选择一个事件话题**，家人据此发问；玩家从手牌打出 1 张卡进入结算。事件呈现同一日常情境在中英语境下的语义差。

| 事件 | 中国经验 | 英国经验 | 示例具象卡 Y | 示例抽象卡 X | 示例融合卡 X·Y | 电话语义差 |
|---|---|---|---|---|---|---|
| **A. 排队（Queue）** | 人情与效率会临时介入秩序 | 先来后到是公共默契 | 《地铁黄线》 | ◻ Structure | 《黄线里的秩序语法》 | 易懂："今天人多不多？" / 难懂："公共秩序压力" |
| **B. 请客 / AA** | 请客是关系表达 | AA 强调边界与公平 | 《收银台争夺》 | ◇ Exchange | 《账单背面的关系坐标》 | 易懂："谁请谁吃饭？" / 难懂："边界透明" |
| **C. 节日翻译（春节 / 圣诞）** | 团圆是行动与义务 | 节日强调氛围与仪式 | 《红包封口》/《壁炉前礼物堆》 | ⚪ Absence | 《节日里的缺席形状》 | 易懂："今年回来吗？" / 难懂："holiday mood" |

#### 4.3.2 出牌结算规则

玩家选定事件后，从手牌打 **1 张卡**：

| 打出卡类型 | 结算 |
|---|---|
| **具象卡 Y** | 获得 1 张"家人回赠具象卡 Y'"（与该事件相关）。可保留（超牌库上限则弃 1）或直接替换 1 张低价值具象卡 |
| **抽象卡 X** | 不获得回赠卡。可执行 1 次"**求解释换卡**"：弃这张 X，抽 1 张同事件的具象卡 Y |
| **融合卡 X·Y** | 二选一：① 抽 1 张具象卡 Y'；② 抽 1 张弱抽象卡 X~（同符号低分值） |
| **不出牌（沉默）** | "算了你忙吧。" → 不产出任何卡 → 短镜头：屏幕另一端的沉默 → 镜头切回房间 |

> 出去的抽象/融合卡在"求解释换卡"或"二选一"中**会被消耗或转换**；具象卡分享则不消耗，只是"展示"给家人后获得回赠。

#### 4.3.3 事件专属加成

每个事件对"对题符号"的卡有额外奖励，鼓励玩家把手牌结构对准事件主题：

- **A（排队）**：打出 `◻ Structure` 相关卡时，额外"看 2 选 1"抽卡。
- **B（请客 / AA）**：打出 `◇ Exchange` 相关卡时，可"交换手牌"1 次（放回 1 张抽 1 张）。
- **C（节日翻译）**：打出 `⚪ Absence` 相关卡时，抽到具象卡后可立即免费尝试 1 次融合（不消耗 5 精力）。

#### 4.3.4 与合成系统的接口

电话换卡是合成系统的**素材补给与转译辅助**：

- 补具象素材（具象 → 回赠 Y'）；
- 在"听不懂"的情况下把抽象卡**回译**为具象卡（抽象 → 求解释换卡）；
- 用融合卡换取最贴合事件主题的素材，提高后续合成成功率。

合成最终仍走形态同构判定：`contour == abstract.sym` → 融合卡 X·Y；不一致 → 强行套壳卡（−10 pts）。

#### 4.3.5 设计内涵

事件 A/B/C 的设计让"和家人渐行渐远"从**抽象的牌库压力**落地为**具体的语义差**：每个事件都有家人能听懂的版本（具象）和家人听不太懂的版本（抽象），融合卡试图在两端之间架桥但仍有损耗。玩家会**意识到**自己的 deck 里这类卡的稀缺，以及"翻译"本身的代价。

### 4.4 上网检索小游戏（独立小游戏）

#### 4.4.0 定位

"上网"不是普通换牌动作，而是一个**独立小游戏环节**：
- 玩家需要一次性投入多张卡，进行一轮"检索 → 拼接 → 提交"的高风险合成。
- 目标是从信息噪声中，换出最终提交作业需要的"最符合目标的复合卡"。
- 结果不稳定：可能大赚，也可能亏掉整手投入。

设计目标：
- 让"上网"成为中后期关键决策，而非顺手操作。
- 强化"信息有真有假"的主题：越想走捷径，越可能被误导。
- 提供可读、可算、可调参的结算模型，便于数值迭代。

#### 4.4.1 进入条件与回合结构

**进入条件**：消耗 **2 精力**（入口费）+ 选择一个作业目标标签（例：`历史+视觉`、`社会+数据`），决定结果池倾向。

**单次小游戏流程**：

1. **选题**：选择检索主题（决定结果池倾向）。
2. **出牌**：从手牌一次性打出 **2–5 张卡**（至少 2 张），即"检索包"（Search Bundle）。
3. **检索判定**：根据主题、出牌结构、噪声值生成信息结果。
4. **合成结算**：产出一张结果卡（或无效卡）并处理损失。
5. **退出**：回到主流程，决定是否用于最终作业提交。

#### 4.4.2 检索包（Search Bundle）出牌规则

- 必须**同时**打出多张卡（非逐张替换）。
- 检索包可包含三类角色：
  - **主题卡**：定义检索方向。
  - **修饰卡**：增加命中或降低风险。
  - **速成卡**：提高收益同时提高污染概率。
- 多卡同投是机制核心：体现"在复杂网络环境中拼凑结论"，让玩家在"投入越多潜在收益越高、爆掉时损失越大"中做抉择。

#### 4.4.3 三种结算结果

| 结果 | 含义 | 产出 | 附加 |
|---|---|---|---|
| **A. 命中（Hit）** | 信息成功对题 | 1 张"高匹配复合卡"（可用于作业提交），匹配度分 `精准` / `可用` 两档 | 可选返还 1 张基础卡（平衡项） |
| **B. 崩溃（Crash）** | 盲信网络被反噬 | 检索包内卡片**全部销毁 / 移出** | 可附带 1 层"信息过载"Debuff（下次上网更危险） |
| **C. 偏航（Drift）** | 国外网站语境偏差 | 1 张"普通抽象卡"（低价值、与当前作业目标弱相关或无关） | 可作为后续合成素材，但本回合相当于"方向跑偏" |

> 偏航结果应保持较常见，才能体现"上网搜到很多看似有内容但不对题的信息"。

#### 4.4.4 概率模型与动态修正

用三段概率直接控制体验：

- `P_hit` + `P_crash` + `P_drift` = 100%

**建议初始值（中风险版本）**：
- `P_hit = 35%`
- `P_crash = 25%`
- `P_drift = 40%`

**动态修正项**：

| 条件 | 修正 |
|---|---|
| 主题匹配度高 | `P_hit +10%` |
| 使用验证类卡 | `P_crash −12%` |
| 噪声值过高 | `P_crash +10%`，`P_hit −5%` |
| 速成卡数量 ≥ 2 | `P_hit +8%`，`P_crash +12%` |

#### 4.4.5 ForeignNoise（跨语境干扰）

为突出"国外网站偏航"这一专属语境问题，引入参数：

- 新增变量 `ForeignNoise`（0–100）
- `ForeignNoise` 高时：
  - 提高 `P_drift`
  - 产出的普通抽象卡更"概念化"，更难直连作业目标

可配套两种反制：
- **本地化检索**：降低偏航概率。
- **术语校对**：把一张普通抽象卡转为"半相关卡"。

#### 4.4.6 玩家可见信息（避免"纯随机感"）

上网界面**直接显示**：

- 本次投入卡数与总风险等级（低 / 中 / 高）。
- 三种结果的当前概率预估（命中 / 全损 / 偏航）。
- 风险来源说明（噪声高、跨语境高、未验证等）。

让玩家认为"我在承担可理解的风险"，而不是被暗骰惩罚。

#### 4.4.7 MVP（最小可玩版本）

先做一版能快速验证手感的：

- 一次必须打出 **3 张** 卡。
- 只产出三种结果：① 目标复合卡；② 全损（失去 3 张）；③ 普通抽象卡。
- 提供一个"验证"开关：开启后 `P_crash` 固定下降。

该 MVP 已完整覆盖核心意图：
> 这是一个独立小游戏；要一次性打多张卡；可能换来最符合目标的复合卡，也可能全亏，或只得到无关抽象卡。

## 5. 卡牌五态

| 类型 | 来源 | 视觉特征 | 使用 |
|---|---|---|---|
| **抽象卡 X** | 课堂捕获 / 上网偏航 | 几何（◻ ⚪ △ ◇ ✚ ⬡） | 作业主题命中 / 电话出牌（求解释换卡） |
| **具象卡 Y** | Telephone（事件回赠）/ Internet（命中）/ Park | 写实形象（🐶 🍊 🐱） | 作业多样性 / 电话出牌（获回赠 Y'）；附隐藏 contour 字段 |
| **融合卡 X·Y** | Homework 合成（contour 匹配）/ Internet 命中（精准） | 由具象演化的几何（"几何狗"） | 双属性，作业里以一抵二 / 电话出牌二选一 |
| **强行套壳卡** (Forced Geometry) | Homework 合成（contour **不**匹配） | 几何框架与具象内容明显错位，视觉上有断裂感（错位边框 + 碰撞色） | 作业提交时每张 −10 pts；无法进行电话出牌 |
| **形态崩塌卡** (Collapsed Form) | 待设计（v0.6 考虑：连续失败合成 / 上网连续 Crash 后触发） | TBD | 预计 −15 pts；代表彻底丧失转译能力 |

**视觉机制即叙事**：外部学术世界给你**空的几何外壳**，自己的生活给你**温暖的具象**。创作的工作是**把后者变成前者的形态**——但消化过程中，温度有没有被保留？强行套壳卡是"形还在，但魂没进去"的状态：几何框架与具象记忆的接缝清晰可见。

## 6. 牌库与作业

### 6.1 牌库恒定

- 总容量 **10 张**
- 初始牌库：8 张母语具象卡（玩家从家乡带来）+ 2 个空位
- 新增卡 → 必须从现有牌库选一张**丢弃 / 保留 / 融合**

### 6.2 作业评分：主题命中 + 多样性

每次作业 brief 指定一个**主题符号**（例如：本周作业主题是 ◻️）。
玩家提交 **4 张卡**（数量可在 playtest 中调整）。

#### A. 主题命中（Theme Match）

提交卡中**至少 3 张**与主题符号相关：

- **抽象卡** → 直接看符号是否匹配
- **具象卡** → 每张具象卡有隐藏的"主题 tag"（湖→ ◻️、橘子→ ⚪、猫→ △……）
- **融合卡** → 因为同时含具象基底和抽象符号，**只要其抽象符号匹配主题**即算命中

#### B. 多样性（Diversity）

4 张卡覆盖的"卡牌类型"数量：

- **4 张同类型** → 单调（最低）
- **2 + 2**（抽象+具象 / 抽象+融合 / 具象+融合）→ 有结构（中）
- **1 + 1 + 2 或 全三类** → 综合（最高）

#### C. 关键规则：融合卡的双属性

**融合卡同时计入"抽象类"和"具象类"两个统计。**

这让融合卡在多样性维度上**以一抵二**，但**不能完全取代两侧**。看下表：

| 提交组合 | 主题 | 多样性 | 教授评价 |
|---|---|---|---|
| 4 抽象 | 命中 | 单调 | "Technically on-brief, but where's the work?" |
| 4 具象 | 失败（无抽象主题符号） | 单调 | "I don't see how this engages with the brief" |
| 2 抽象 + 2 具象 | 命中 | 中 | "Some engagement, but feels collaged" |
| 1 抽象 + 1 具象 + 2 融合 | 命中（多个） | 高 | "**A clear voice working through the brief**" ← 最高分 |
| 4 融合卡 | 命中 | **单调（仍是单一类型）** | "Strong voice but underdeveloped — where's the rigor?" ← **故意压低** |

**全融合策略被故意压低**——理由是缺少 raw 抽象/具象的对照。这避免"全融合最优"的策略坍缩。

#### D. 失败卡扣分（v0.4 新增）

提交手牌中的失败卡直接从评分中扣除：

| 失败卡类型 | 每张扣分 |
|---|---|
| **强行套壳卡**（Forced Geometry） | −10 pts |
| **形态崩塌卡**（Collapsed Form，v0.6） | −15 pts |

失败卡不影响 theme match 判定（仍可命中主题），但会拉低最终评级。理性策略是：宁可不交失败卡，也不要凑数。

### 6.3 brief 模糊机制

如果听课捕获不足，brief 的主题符号会有部分以 "?" 显示——玩家被迫赌：

- 猜对了 → 主题命中加分
- 猜错了 → 主题命中失败，且全部命中分数清零

这对应"没听懂作业要求就开始做"的真实赌博感。

---

## 7. 三结局

通关时由**牌库构成**决定，**无优劣关系**：

- **Clean Slate**（同化）：母语具象卡 ≤ 2，无融合卡 → 作业评价高，**电话连续失败**。结尾镜头：家里来电话，"我现在不太方便，晚点打回去"，挂断，flash 闪过被丢弃的母语卡。
- **Untranslated**（坚守）：母语具象卡 ≥ 6，抽象/融合卡 ≤ 2 → **作业勉强及格，电话也接不下去**。结尾镜头：自己说母语清晰，走廊里同学说话仍是乱码；电话铃响，犹豫片刻，没接。
- **Fluent in Two Tongues**（融合）：融合卡 ≥ 3 → 双语流利，作业和电话都能维持，但**这条路最累**。结尾镜头：熬夜蒙太奇 + 提交作业 + 揉太阳穴 + 接起电话用母语笑着说话。

三个结局演出长度、配乐、镜头语言**等量齐观**。**不奖励"正确路线"**。

## 11. 申请文档配套（重要）

提交研究生作品集时，**配套设计文档比游戏本身更值钱**。文档应主动讨论：

1. **个人经验 → 机制的转译过程**
   "国际学生作业焦虑"如何具体转化为"30 秒乱码倒计时 + 牌库恒定容量 + 电话失败机制 + 上网检索高风险"？为什么是这些机制而不是别的？

2. **与参考作品的关系**
   - 与 *Signs of the Sojourner*：保留了牌库演化（成长伴随失去），**舍弃了边缘符号匹配**——为什么？因为我的卡牌视觉不支持，且我的核心张力来自 deck 构成的双向夹击，不来自单次出牌的拼接技巧
   - 与 *Chants of Sennaar*：保留了在不确定中构建语义理解，舍弃了主动解谜的从容感

3. **失败的中间版本**
   - "拼图比喻"为什么不够好？
   - 为什么放弃了 21 天 roguelike 框架？
   - 为什么把超市并入上网？
   - 为什么把"全 X 卡 → 低分"的简单逻辑改成"主题 + 多样性"双维度？（提示：策略坍缩问题）
   - 为什么电话从"分享卡"重设计为事件 A/B/C？（提示：让"中英语义差"从抽象命题变成可玩内容）
   - 为什么上网从"顺手换牌"升级为高风险检索小游戏？（提示：让"信息有真有假"对应到玩家行为成本）

> 展示 critical design literacy 是 MA 申请最值钱的部分。**敢于在文档里写"我砍掉了什么、为什么砍"，比假装一切都设计完美更显成熟**。

## 12. 设计理由：为什么不用概率失败，而用形态同构

v0.3 的合成系统是"不匹配 = 拦截（变灰不可选）"。v0.4 改为"不匹配 = 可选，但产出失败卡"。这是一个有意的设计方向选择，理由如下：

**为什么不用概率失败（random chance）？**

概率失败（"有 30% 概率合成失败"）在机制上诚实，但在叙事上是空洞的。玩家失败时没有任何可读信息：既不知道自己做错了什么，也没有办法学习和改进。这与游戏的核心主题（在不确定中构建理解）相悖——游戏应该给玩家留下可解读的残骸，而不是随机噪声。

**为什么不用拦截（变灰不可选）？**

拦截是最安全的设计：它告诉玩家"这条路不通"，防止浪费资源。但它也关上了一个叙事可能性：玩家永远不会亲眼看到强行套壳的结果是什么样的。失败卡的视觉（错位框架 + 碰撞色）是游戏叙事的一部分——"形还在，但魂没进去"这个状态需要被玩家看到、感受到，而不是被提前阻止。

**形态同构的叙事逻辑：**

具象卡的 contour（形态轮廓）代表它在视觉/结构层面的"潜在抽象形"。抽象卡的 sym 代表学术话语赋予的几何外壳。只有当两者一致，消化才真正发生；否则只是强行套上了一个壳——这正是国际学生在学术写作中常见的困境：用正确的词，但意思没有真正进入形式。

**保留"故意选错"的可能性：**

有经验的玩家可能会故意制造强行套壳卡——作为牌库管理的一种策略（消耗掉不想要的卡，换一个占位的失败卡）。这是一种合理的边缘策略，设计上不应该拦截它。

**v0.5 的延伸：上网用概率三分，电话用确定分支——为什么并存？**

合成保持"形态同构"（确定结果）是因为它是**核心创作行为**——叙事必须可读。
上网用三段概率（`P_hit / P_crash / P_drift`）是因为它要表现**信息环境本身的不确定**——但玩家可见所有概率与风险来源，不是暗骰惩罚。
电话用"卡类型 → 分支结算"（确定结果）是因为它要表现**语义差是结构性的，不是运气问题**——同一事件下，具象、抽象、融合各自走可解读的路径。

三种结算风格各自承担不同主题：合成 = 转译能力；上网 = 信息噪声；电话 = 语义差。

## 13. 待 v0.6 考虑的事项（仅记录，不做）

- **形态崩塌卡（Collapsed Form）的触发条件**：连续失败合成（如 3 次强行套壳）或上网连续 Crash 后，下次合成/检索无论是否匹配都产出崩塌卡？还是作为一种特殊的高代价行动？
- **contour 与 tag 的独立配表**：目前 contour = tag（默认），后期可以让具象卡的 contour 不等于 tag，制造更复杂的策略空间（某张卡在主题上命中 ◻，但在形态上属于 ⚪）。
- **电话场景的反向冲突**：家人是否能"感知到"牌库里有失败卡？失败卡是否会影响电话事件 A/B/C 的对话内容？
- **事件 A/B/C 的扩展**：是否新增事件 D / E（例：医疗叙述、租房沟通），覆盖更多中英语境差；或允许同一事件多次重访产出递进式回赠卡。
- **上网"信息过载"Debuff 的具体作用**：Crash 附带的 Debuff 在下一次检索如何具体生效？提高 `P_crash`？锁定一个槽位？还是污染下一次抽到的具象卡？
- **本地化检索 / 术语校对**作为可解锁的反制卡：是否作为电话事件回赠或合成产出？
