# GDD v0.2 —《乱码与卡牌 / Static & Signal》

> **v0.2 修订说明**
> 基于 v0.1 实现过程中的反馈：原文档机制层叠太多，HTML 原型难以呈现。
> v0.2 围绕"每日精力分配"重组结构，砍掉非必要复杂度，优先保证 vertical slice 可实现。
> 主题、情感底色、卡牌三态视觉、三结局结构维持不变。

---

## 1. 核心概念（30 秒版）

你是一个刚到英国的研究生。每天 **15 点精力**。

上午必须去上课——导师讲话是逐词滚过的乱码，**30 秒实时倒计时**，你只能捕获其中几个词。
下午地图打开，你自己决定剩余精力怎么花：去公园观察、上网搜索、给家里打电话，或者回房间把素材合成成卡牌。
每周交一次作业，用卡牌出链。
听课捕获越多 → 外语卡越多；打电话越多 → 母语卡越多。
**牌库容量恒定。有些东西必须被丢掉。**

---

## 2. 日循环结构

每天两段：

```
[早上 09:00] ─── 强制上课（30 秒）
                       │
                       ▼
              ─── 地图打开 ───
                       │
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
    [Apartment]     [Park]          [School]
                       │
                精力耗尽 / 主动睡觉
                       │
                       ▼
                  [次日 09:00]
```

**精力归零 或 玩家主动"睡觉"** → 推进到下一天。

## 3. 上午：听课（强制 / 30 秒）

```
┌──────────────────────────────────────┐
│  [⏱ 00:30]                            │
│                                       │
│   👨‍🏫    ┌──────────────────────┐    │
│   导师    │ @#￥%……  &*￥#@  ……%& │   │
│           │  ￥#@  *&……%   *@%   │    │
│           └──────────────────────┘    │
│                                       │
│                          🧑 ●●●●●     │
│                          (玩家精力)    │
└──────────────────────────────────────┘
```

**机制**：
- 进入课堂自动消耗 **3 点精力**（出场费，无法回避）
- 30 秒倒计时内，乱码词在对话气泡里**逐词滚过**
- 玩家点击某个乱码词 → "捕获"它 → 消耗 **1 点精力**
- 倒计时结束 或 玩家精力耗尽 / 主动跳过 → 课程结算
- 被捕获的乱码进入"**未翻译素材池**"，还不能直接用

**典型一节课**：玩家捕获 5–8 个词，消耗 8–11 点精力，下午剩 4–7 点。

### 3.1 一个待 playtest 的关键设计问题

> **玩家凭什么知道该捕获哪个词？**

三个候选方案，倾向方案 C：

- **方案 A（纯随机）**：所有乱码词等价。玩家瞎猜。→ 破坏玩家能动性，不推荐。
- **方案 B（明示标注）**：重要词高亮显示。→ 破坏"听不懂"的体验内核。
- **方案 C（弱暗示）**：部分乱码词有**微弱视觉差异**（略大、略亮、出现略慢），但暗示**不完全可靠**——有时高亮的也是垃圾词。**玩家在游戏过程中逐渐学会读这些暗示**，命中率提升不是靠角色升级，而是**靠玩家自己习得模式**。

方案 C 让"语言习得"这件事真的发生在玩家身上，而不是被参数模拟出来。但需要 playtest 调暗示强度。

## 4. 下午：地图系统

地图层级（参考玩家草图）：

```
[Map]
 │
 ├── 🏫 School         上午已访问，下午可选择性访问
 │
 ├── 🏠 Apartment      回家
 │    ├── 📞 Telephone     与家人通话
 │    ├── 💻 Internet      上网搜索灵感
 │    └── 📐 Homework      合成卡牌 & 做作业
 │
 └── 🌳 Park           户外
      ├── 🌊 Lake          安静、自然、内省
      └── 🌿 Lawn          社会、活跃、人群
```

### 4.1 各地点行为

| 地点 | 精力 | 产出 | 备注 |
|---|---|---|---|
| 📞 Telephone | **0** | 1 张具象卡（家乡 / 童年 / 母语） | 唯一免费行为 |
| 💻 Internet | 2 | 1 张具象卡（数字图像、网络文化） | 信息量大但浮于表面 |
| 📐 Homework — 合成 | **5** | 把 1 张抽象卡 + 1 张具象卡 → 1 张融合卡 | 最贵，需满足相似性条件 |
| 📐 Homework — 出牌 | 0 | 提交作业 | 仅在作业截止日可用 |
| 🌊 Lake | 2 | 1 张具象卡（自然、水、安静意象） | |
| 🌿 Lawn | 2 | 1 张具象卡（人物、社交、城市意象） | |

### 4.2 精力账本（典型一日）

```
开局:                  15 点
强制上课:              -3 点  → 12 点
课堂捕获 5 词:         -5 点  →  7 点
回家翻译 2 个素材:     -4 点  →  3 点
打电话:                 0 点  →  3 点
去公园 Lake:           -2 点  →  1 点
睡觉
```

打电话的免费性是设计上的**情感锚点**：母语世界永远不收费，但它也不直接产出"作业素材"——具象卡需要被合成才能出现在作业链条里。

## 5. 卡牌三态

| 类型 | 来源 | 视觉特征 | 使用 |
|---|---|---|---|
| **抽象卡** | 课堂捕获 → Homework 翻译 | 几何（◻️ ⚪ △ ◇） | 直接可用 |
| **具象卡** | Telephone / Internet / Park | 写实形象（🐶 🍊 🐱） | 直接可用，但 Voice 强、Legibility 弱 |
| **融合卡** | Homework 合成（抽象 + 具象） | 由具象演化的几何（"几何狗"） | 双属性，链条最灵活 |

**视觉机制即叙事**：外部学术世界给你**空的几何外壳**，自己的生活给你**温暖的具象**。创作的工作是**把后者变成前者的形态**——但消化过程中，温度有没有被保留？

> 美术风格建议：**手绘拼贴**。母语卡线条手绘感强，外语卡矢量精度高，融合卡杂交。这个细节是单人 indie 最容易出彩的地方。

## 6. 牌库与作业

### 6.1 牌库恒定

- 总容量 **12 张**
- 初始牌库：8 张母语卡（具象，玩家从家乡带来的）+ 4 个空位
- 新增卡 → 必须从现有牌库选一张**丢弃 / 保留 / 融合**（v0.1 第 4.4 节的核心机制保留）

### 6.2 作业评分（三轴）

每周交一次作业 = 出 3-7 张卡组成链条，相邻卡符号必须匹配（可强行接，扣分）。

| 轴 | 由什么决定 |
|---|---|
| **Legibility** | 链条中抽象/具象卡/融合卡的占比 + 是否命中 brief 要求的符号 |
| **Voice** | 链条中母语具象卡的占比 + 母语符号是否在关键位置 |
| **Coherence** | 卡牌相邻符号匹配的质量；强行连接扣分 |

三轴互相牵制。**全外语卡 → Voice 为零；全母语卡 → Legibility 为零；混合 → Coherence 容易翻车**。

> 此外：如果听课捕获不足，brief 中"必需符号"会有部分以 "?" 显示——玩家被迫赌。

---

## 7. 三结局

通关时由**牌库构成**决定，**无优劣关系**：

- **Clean Slate**（同化）：母语卡 ≤ 2，无融合卡 → 流畅但失声。结尾镜头：家里来电话，"我现在不太方便，晚点打回去"。
- **Untranslated**（坚守）：母语卡 ≥ 6 → 异域风情但勉强毕业。结尾镜头：自己说母语清晰，走廊里同学说话仍是乱码。
- **Fluent in Two Tongues**（融合）：融合卡 ≥ 3 → 双语流利，但更累。结尾镜头：熬夜蒙太奇 + 提交作业 + 揉太阳穴。

三个结局演出长度、配乐、镜头语言**等量齐观**。不奖励"正确路线"。

## 8. 总流程

| 版本 | 长度 | 用途 |
|---|---|---|
| **Vertical Slice** | 1 周（7 天）+ 1 次作业 | 申请用 demo，约 15 分钟 |
| **完整版** | 3 周（21 天）+ 3 次作业 + 终评 | 约 40–50 分钟 |

vertical slice 必须呈现：
- ✅ 完整的日循环（早上听课 + 下午地图）
- ✅ 至少 1 次作业的三轴评分
- ✅ 至少 1 次"丢弃 / 保留 / 融合"的牌库决策
- ✅ 至少 1 次合成（让玩家亲手做出 1 张融合卡）

## 10. v0.1 → v0.2 简化追溯

为让 demo 可实现，v0.2 移除或推迟：
- 注意力池**动态扩容**机制 → 简化为固定 15 点/天
- 5 个**语义维度**（Personal / Conceptual / Material / Reference / Context）→ 保留 5–6 个抽象符号但不细分语义层
- **超市**作为独立场景 → 合并入"上网搜索"
- **多周长流程** → vertical slice 仅做 1 周
- **记忆卡 / 观察卡** 子类型区分 → 简化为统一"具象卡"

如 vertical slice 验证机制成立，可在完整版重新引入。

## 11. 申请文档配套（重要）

提交研究生作品集时，**配套设计文档比游戏本身更值钱**。文档应主动讨论：

1. **个人经验 → 机制的转译过程**
   "国际学生作业焦虑"如何具体转化为"30 秒乱码倒计时 + 牌库恒定容量"？为什么是这个机制而不是别的？

2. **与参考作品的关系**
   - 与 *Signs of the Sojourner*：保留了符号匹配 + 牌库演化（成长伴随失去），舍弃了 NPC 关系系统、长流程
   - 与 *Chants of Sennaar*：保留了在不确定中构建语义理解，舍弃了主动解谜的从容感

3. **失败的中间版本**
   "拼图比喻"为什么不够好？为什么放弃了 21 天 roguelike 框架？为什么把超市并入上网？

> 展示 critical design literacy 是 MA 申请最值钱的部分。**敢于在文档里写"我砍掉了什么、为什么砍"，比假装一切都设计完美更显成熟**。
